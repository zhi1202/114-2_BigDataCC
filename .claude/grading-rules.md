# 批改標準與評分細則

> 用途：Claude 批改 PR 和作業時的參考標準

## 一、平時作業批改（20%）

### PR 檢查清單

1. **PR 標題格式**：`學號_姓名_weekXX` → 格式不符先請修正
2. **檔案位置**：作業是否在正確的 `weekXX/` 目錄下
3. **檔案完整性**：是否完成所有要求的題目
4. **內容正確性**：指令輸出、程式碼是否正確
5. **commit 訊息**：是否有描述性，非 "update" / "fix" 等無意義訊息

### 評分等級

| 等級 | 分數 | 條件 |
|------|------|------|
| A | 95 | 全部正確 + 有額外探索或自主學習紀錄 |
| A- | 90 | 全部正確 |
| B+ | 85 | 大致正確，1–2 處小問題 |
| B | 80 | 大致正確，有 commit 訊息品質問題 |
| C+ | 75 | 完成 80% 以上，有錯誤但展現努力 |
| C | 70 | 完成 60% 以上 |
| D | 60 | 僅完成部分，缺漏嚴重 |
| F | 0 | 未繳交或明顯抄襲 |

### 遲交處理

- 截止後 1 天內：扣 10 分
- 截止後 2–3 天：扣 20 分
- 截止後 3 天以上：扣 30 分
- 超過 1 週：不接受

### 抄襲判定

- 兩份以上 PR 的指令輸出完全相同（含錯誤）→ 標記可疑
- commit 時間集中在截止前 10 分鐘 + 內容與他人高度相似 → 標記可疑
- 最終判定由教師確認

## 二、小考評分（20%）

- 選擇題：每題固定分數
- 簡答題：按要點給分，部分正確給部分分數
- 實作題：能正確執行給滿分，有小語法錯但邏輯正確給 80%
- 四次小考平均計算

## 三、期中考評分（20%）

- 評分標準同小考
- 額外注意：指令實作題必須考慮邊界情況

## 四、期末專題評分（20%）

詳見 README.md 的專題評分四維架構。

### AI 評分腳本檢查項目

1. Commit 次數與頻率
2. Commit 時間分布（是否趕工）
3. 程式碼貢獻行數
4. Commit 訊息品質
5. 檔案覆蓋範圍
6. 專題結構完整度
7. Docker 部署可行性

### AI 評分與組內互評交叉比對規則

- AI 評分 A + 組內互評低 → 教師個別確認
- AI 評分 C + 組內互評高 → 教師個別確認
- 兩者一致 → 直接採用

## 五、課堂參與（20%）

- 出席：曠課 4 次以上建議停修
- 課堂提問與回答
- 課堂練習參與度

## 六、回饋語氣指引

Claude 批改時遵循：
- 先肯定做得好的部分
- 用「建議」而非「錯誤」的語氣指出問題
- 提供具體的修正方向
- 鼓勵額外探索
- 不使用負面或嘲諷語氣
- 中文回饋為主，技術術語保留英文
